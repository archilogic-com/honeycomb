# Colors

## Usage

### Tailwind classes

You can use these colors as classes generated by Tailwind CSS. They are automatically shared by all color-driven utilities, like `textColor`, `backgroundColor`, `borderColor`, and more.

_Each story is named in the sidebar after the relevant class name._

Additionally, there are `transparent` and `current` color classes.

### JavaScript

The color palette map can also be imported directly in JS from the `@archilogic/honeycomb/colors` entry point. The export is a map of color names to CSS color strings:

```ts
import { colorMap } from '@archilogic/honeycomb/colors'
```

## Deprecated

These color names are static hex values and will be removed in the next major version.
Use the semantic `primary-*` names instead — they follow the active theme and support runtime overrides.
Use the old names only where a static color is required (e.g. canvas / WebGL rendering).

- `mediumblue` → `primary`
- `blueribbon` → `primary-hover`
- `darkblue` → `primary-active`
- `zurich` → `primary-subtle`
- `zurich48` → `primary-subtle-alpha`
