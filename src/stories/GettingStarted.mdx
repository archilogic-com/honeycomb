# Getting Started

## Installation

```
npm i @archilogic/honeycomb
```

This library requires several peer dependencies: `tailwindcss` (v4+), `vue`, and `@headlessui/vue`.
They will be installed automatically unless there is a version conflict.
Please refer to `package.json` to determine a compatible version.

## Configuring Tailwind CSS

This library relies on Tailwind CSS v4 with the CSS-first configuration model.
No `tailwind.config.js` or `postcss.config.js` files are needed.

### Vite plugin

Install the Tailwind CSS Vite plugin:

```
npm install -D @tailwindcss/vite tailwindcss
```

Add it to your `vite.config.ts`:

```ts
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss(), vue()]
})
```

### CSS entry point

Create a CSS file that imports Honeycomb's Tailwind configuration:

```css
/* e.g. src/tailwind.css */
@import '@archilogic/honeycomb/tailwind';
```

Honeycomb automatically registers its own `dist/` directory as a Tailwind source.
If your app uses other packages with Tailwind classes, add `@source` directives for them:

```css
@import '@archilogic/honeycomb/tailwind';
@source "../node_modules/@archilogic/app-kit/dist/**/*.{js,css}";
```

**Why is this a CSS `@import` and not a JavaScript import?** `@archilogic/honeycomb/tailwind` is Tailwind _source_ — design tokens (`@theme`), cascade layers, and component rules — not a prebuilt stylesheet. Your app's own Tailwind build compiles it, generating only the utilities you actually use and letting you override tokens. That is why it must be pulled in from a `.css` file rather than imported in JS, and why there is no separate CSS bundle: this one line ships every style Honeycomb needs.

### Import CSS

Import the CSS entry point in your app:

```ts
// e.g. in main.ts
import './tailwind.css'
```

### Using `@apply` in Vue scoped styles

If you use `@apply` with Honeycomb utilities in Vue `<style scoped>` blocks,
add `@reference` at the top of the style block:

```vue
<style scoped>
@reference "@archilogic/honeycomb/tailwind";
dt {
  @apply body-sm-600;
}
</style>
```

## Component Registration

Registering all available components globally:

```js
import { registerAllComponents } from '@archilogic/honeycomb'

const app = createApp(MyApp)

registerAllComponents(app)
```

To import and register a subset of components, use one of the following ways.

Import from the main package export:

```js
import { AButton } from '@archilogic/honeycomb'
```

Or import from the dedicated `components` export:

```js
import { AButton } from '@archilogic/honeycomb/components'
// Or import all components:
import * as components from '@archilogic/honeycomb/components'
```

Then register locally:

```js
export default defineComponent({
  name: 'MyComponent',
  components: { AButton }
})
```

Or globally:

```js
const app = createApp(MyApp)
app.component('a-button', AButton))
```

## Composables

The library exports some composable functions as well.
They can be imported in the similar way as components.

Or from the main package export:

```js
import { useMultiSelect } from '@archilogic/honeycomb'
```

Either from the dedicated `composables` export:

```js
import { useMultiSelect } from '@archilogic/honeycomb/composables'

// Or import all composables:
import * as composables from '@archilogic/honeycomb/composables'
```

## Upgrading

### From 3.1 to 3.2

Honeycomb no longer ships a prebuilt CSS bundle. If your app imported the compiled
stylesheet directly, delete that import — neither path exists in 3.2:

```ts
// no longer exists in 3.2 — delete
import '@archilogic/honeycomb/dist/style.css'
import '@archilogic/honeycomb/css'
```

Component styles now arrive through the Tailwind preset you already import in your CSS
entry (`@import '@archilogic/honeycomb/tailwind'`). If you do not yet import the preset that
way, follow [Configuring Tailwind CSS](#configuring-tailwind-css) above — that single import
is now the only stylesheet Honeycomb needs.
